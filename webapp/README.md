# 스마트피트니스 근력운동 건강영향 — 자동 분석 웹앱

논현노인종합복지관의 **근력운동 기기 로그 + InBody 검사 데이터**를 마스터 스프레드시트에서 읽어,
[연구 설계서](../스마트피트니스_근력운동_건강영향_연구설계.md)의 **RQ1~RQ4 분석과 대시보드를 자동으로** 그려주는 웹 애플리케이션입니다.

- **백엔드 없음** — 순수 HTML/CSS/JS 정적 앱. 브라우저에서 스프레드시트 CSV를 직접 읽어 분석.
- **실시간 누적** — 스프레드시트에 데이터가 쌓일수록, 새로고침만 하면 최신 분석으로 갱신.
- **배포 후 링크만 연결** — 앱 화면의 `⚙ 설정`에 스프레드시트 CSV 링크 2개를 붙여넣으면 끝.

---

## 1. 폴더 구조

```
webapp/
├── README.md          ← (이 문서) 설계·배포·사용 안내
├── index.html         ← 대시보드 화면
├── style.css          ← 스타일(고령자 친화: 큰 글씨·고대비·반응형·다크모드)
├── config.js          ← 스프레드시트 링크 기본값(선택). 화면 설정으로도 입력 가능
└── js/
    ├── csv.js         ← CSV 파서
    ├── stats.js       ← 통계(회귀·백분위·분절회귀·MDC)
    ├── data.js        ← 데이터 로드·컬럼 매핑·조인·파생변수
    ├── charts.js      ← 차트 렌더링(Chart.js)
    ├── demo.js        ← 데모용 가상 데이터(링크 없을 때 미리보기)
    └── app.js         ← 화면 구성·탭·오케스트레이션
```

> 외부 의존성은 차트 라이브러리(Chart.js) 하나뿐이며 CDN으로 불러옵니다. 통계 계산은 모두 자체 구현이라 별도 서버가 필요 없습니다.

## 2. 데이터 준비 (마스터 스프레드시트)

하나의 구글 스프레드시트에 **탭 2개**를 둡니다. 연계 키(`member_key`)는 두 탭에서 **동일한 형식**으로 넣어 주세요(사용자가 통일 제공 예정).

### 탭 A. `운동로그` (기기 로그 — 세트마다 1행, 계속 append)
| 컬럼 | 필수 | 설명 |
|------|:---:|------|
| `member_key` | ✅ | 회원 식별키(두 탭 공통 형식) |
| `sex` | ✅ | 성별 `M`/`F` |
| `birth` | | 생년월일(`YYYY-MM-DD` 등). 연령 계산용 |
| `exercise_dt` | ✅ | 운동 일시 |
| `machine` | ✅ | 운동종류(기기). 예: `Seated Chest Press` |
| `max_force` | ✅ | 최대힘 — **근력 1차 지표** |
| `avg_force` | | 평균힘 |
| `reps` | | 횟수 |
| `volume` | | 볼륨 |

### 탭 B. `인바디` (InBody 검사 — 측정마다 1행)
| 컬럼 | 필수 | 설명 |
|------|:---:|------|
| `member_key` | ✅ | 회원 식별키(운동로그와 동일 형식) |
| `sex` | | 성별 |
| `birth` 또는 `age` | | 연령 계산/표기 |
| `test_date` | ✅ | 측정일 |
| `SMI` | | 골격근지수(근감소증 핵심) |
| `SMM` | | 골격근량 |
| `PBF` | | 체지방률 |
| `VFL` | | 내장지방레벨 |
| `BMI` `weight` `BMR` `BMC` | | 체성분/대사/골 |

> **컬럼명 유연 인식**: 한글/영문, 그리고 InBody 원본 헤더(`96. SMI`, `34. PBF (Percent Body Fat)` 등)도 자동 인식하도록 되어 있습니다. 그래도 가능하면 위 표의 이름으로 정리하는 것을 권장합니다.

## 3. 스프레드시트 → CSV 링크 만들기 (구글 시트)

가장 안정적인 방법은 **"웹에 게시(Publish to the web)"** 입니다. (CORS 문제 없이 브라우저에서 읽힘)

1. 구글 시트에서 **파일 → 공유 → 웹에 게시**
2. **링크** 탭 → 게시 대상에서 **`운동로그` 탭** 선택, 형식 **쉼표로 구분된 값(.csv)** 선택 → **게시**
3. 생성된 URL 복사 (형태: `https://docs.google.com/spreadsheets/d/e/…/pub?gid=0&single=true&output=csv`)
4. 같은 방식으로 **`인바디` 탭**도 CSV로 게시하고 URL 복사
5. 웹앱 화면의 **⚙ 설정**에 두 URL을 붙여넣고 저장

> 두 URL을 `config.js`에 미리 넣어두면 배포 즉시 자동 로드됩니다(3장 하단 예시). 화면 설정 입력은 브라우저에 저장(localStorage)되어 다음에도 유지됩니다.

`config.js` 예시:
```js
window.APP_CONFIG = {
  EXERCISE_CSV_URL: "https://docs.google.com/spreadsheets/d/e/…/pub?gid=0&single=true&output=csv",
  INBODY_CSV_URL:   "https://docs.google.com/spreadsheets/d/e/…/pub?gid=12345&single=true&output=csv",
};
```

## 4. 배포 방법 (택1)

- **로컬 확인**: `webapp` 폴더를 열고 `index.html` 더블클릭. (일부 브라우저는 로컬 파일의 외부 CSV fetch를 막을 수 있어, 아래 정식 배포 권장)
- **GitHub Pages**: 저장소에 `webapp` 내용을 올리고 Pages 활성화 → 공개 URL 생성.
- **Netlify / Vercel**: 폴더를 드래그 앤 드롭 배포. 가장 간단.
- **구글 사이트/사내 웹서버**: 정적 파일 그대로 업로드.

배포 후 나온 공개 URL을 즐겨찾기 해두면, 언제든 접속 시 스프레드시트 최신 데이터로 분석이 갱신됩니다.

## 5. 화면(탭) 구성 — 연구질문 대응

| 탭 | 내용 | 대응 |
|----|------|------|
| **개요** | 참여 인원·세션수·월별 추이, 근감소증 위험군 비율, 데이터 상태 | 운영 현황 |
| **RQ1 근력수준** | 기기×성별×연령대 최대힘 백분위표·중앙값 막대, 개인 위치 조회 | RQ1 |
| **RQ2 근력추이** | 개인별 근력 기울기 분포, 향상/유지/감소 비율, 빈도군별 비교 | RQ2 |
| **RQ3 정체기** | 운동 경과기간 대비 근력 곡선 + 분절회귀 변곡점(정체 시작점) | RQ3 |
| **RQ4 운동빈도** | 연간 세션수 구간별 근력 변화량 비교, 유의 임계 빈도 탐색 | RQ4 |
| **체성분** | 운동 부위(상체/하체)별 InBody 변화 비교, SMI/PBF/VFL 추이, 유산소 병행군 vs 비병행군 | 부목적 |

### 분석 대상 선택 — 부위별·병합 3종 + 개별 기기
상단 **분석 대상(부위·기기)** 드롭다운에서 다음을 고릅니다.
- **종합(부위별·병합)**: `전체 기기(병합)` · `상체 전체` · `하체 전체` — 여러 기기를 묶어 하나의 근력 지수로 분석
- **개별 기기**: Leg Press, Seated Chest Press … (기기명 옆에 상체/하체 부위 표시)

> **병합 분석의 원리**: 기기마다 힘의 절대 크기가 달라(레그프레스 ≫ 체스트프레스) 그대로 합치면 왜곡됩니다. 그래서 종합 그룹은 각 기기의 힘을 **기기별 표준화(z점수)** 로 바꾼 뒤 합산합니다. 값의 의미는 `0 = 해당 기기 평균, +1 = 1표준편차 위`이며, 부위 전체의 상대적 근력 수준·추이를 비교할 수 있습니다. 개별 기기를 고르면 원래 단위(최대힘)로 표시됩니다.

### 인바디 부위별 분석
체성분 탭은 회원을 **운동 로그의 상체/하체 세션 비율**로 `상체 위주 / 하체 위주 / 상하체 균형`으로 분류(상체>하체×1.5 → 상체 위주 등)하여, 부위별로 InBody 변화(SMI·골격근량·체지방률·내장지방)를 비교합니다. 탭 내 **인바디 대상(운동 부위)** 드롭다운으로 특정 부위 참여자만 필터링해 SMI·체지방 변화를 볼 수 있습니다.

## 6. 분석 로직 요약 (js/stats.js·data.js)

- **부위 분류**: 기기명을 키워드로 상체/하체/몸통/유산소로 자동 분류(`js/data.js`의 `bodyRegion`). 실제 기기명에 맞춰 키워드를 보완할 수 있습니다.
- **표준화(z점수)**: 병합 분석 시 각 기기의 `max_force`를 `(값−기기평균)/기기표준편차`로 변환해 서로 다른 기기를 합산 가능하게 함.
- **백분위(RQ1)**: 성별×연령대 그룹에서 측정값(개별=최대힘, 병합=z)의 P10/25/50/75/90.
- **개인 근력 기울기(RQ2)**: 개인별 `측정값 ~ 시간(일)` 단순 선형회귀 기울기. 측정 2회 이상만.
- **정체기(RQ3)**: X=운동 시작 후 경과(개월), Y=개인 내 표준화 근력. **연속 분절회귀**(hinge 기저)로 변곡점 추정 → 이후 기울기≈0이면 정체.
- **빈도 임계(RQ4)**: 개인 연간 세션수로 구간화(<12/12–35/36–71/72+) 후, 구간별 근력 변화량(마지막−처음) 비교 + 최저군 대비 웰치 t검정(참고용).
- **근감소증 플래그**: AWGS 2019 참고 SMI 남<7.0·여<5.7.

> 반복측정·불규칙 시점 자료의 정밀 추론(선형혼합모형)은 브라우저 요약분석 이후, 필요 시 R/Python으로 확장하도록 설계서에 안내되어 있습니다. 웹앱은 **상시 모니터링·기술통계·추세 근사**를 담당합니다.

---

## 7. 데이터가 없을 때

설정에 링크가 없으면 앱은 **데모 데이터**로 화면을 채워 동작을 미리 보여줍니다(상단에 "데모 모드" 배지 표시). 실제 링크를 넣으면 즉시 실데이터로 전환됩니다.

## 8. 개인정보 유의

- 분석·공유 화면에는 이름·연락처를 두지 말고 `member_key`만 사용하세요.
- 공개 게시(Publish to web) 링크는 URL을 아는 사람이 접근 가능하므로, 민감정보(이름·연락처·주소)는 스프레드시트의 공개 탭에 넣지 마세요.
